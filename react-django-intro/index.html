React & Django intro

Front-end and back-end foundations for modern web apps

What is React?

A JavaScript library for building user interfaces, maintained by Meta (Facebook).

  • Component-based: build UIs from reusable pieces
  • Declarative: describe what the UI should look like; React updates the DOM
  • Works with a single HTML root and a bundler (e.g. Vite, Create React App)

React in one slide

You write components that return JSX. State drives re-renders.

function Hello({ name }) {
  return <h1>Hello, {name}!</h1>;
}

Use useState, useEffect, and hooks for interactivity.

What is Django?

A high-level Python web framework for building secure, scalable back-ends quickly.

  • MVC-style: models, views, templates (plus URL routing)
  • ORM, admin, auth, and batteries included
  • REST APIs via Django REST framework

Django in one slide

Define models, views, and URLs. Templates or JSON for responses.

# views.py
def hello(request):
    return JsonResponse({"message": "Hello"})

Run with python manage.py runserver.

When to use which?

  • React: rich, dynamic UIs; SPAs; mobile (React Native).
  • Django: server-rendered pages, APIs, admin, content sites, auth and data layer.
  • Together: Django serves API + static build; React consumes API and handles the UI.
